# Questions > Why is accessibility so important? A: It improves the user experience (especially for those using screen readers/have difficulty distinguishing color), it improves search engine optimization, and it's easier for future devs to read the source code. > What are some forms of accessibility as it relates to web development? A: Some forms are using proper semantics, using tables only for data, avoiding nested drop-downs that need fine motor skills to operate, making text/background clear with high contrast, etc. > In your own words, what does semantic mean? A: Semantics involves studying meaning, symbols, etc. in a language, and how to communicate most effectively using that information. > What is semantic HTML? A: Semantic HTML is descriptive, and tags are used that best describes the nested content. > What are some common accessibility issues your end users might have? What are some steps you would take to make their experience on your site better? A: Low contrast for text/background can make it difficult to see, so make sure text is contrasted, bold, etc. Also avoid auto-playing media, and UI components that are difficult to use without fine motor skills.
